## 데코레이터  
~~~
데코레이터 패턴은 주어진 상황 및 용도에 따라 어떤 책임을 덧붙이는 패턴으로, 
기능 확장이 필요할 때 서브클래싱 대신 쓸 수 있는 유연한 대안이 될 수 있다.
~~~
위키피디아의 설명이다. 즉, 두 객체의 결합을 통해서 기능을 유연하게 확장할 수 있게 해주는 패턴을 의미한다.  
예를 들어, 기존에 작성된 클래스를 기반으로 어떤 기능을 확장해야할 경우 기존 클래스를 상속받는 자식클래스를 새로 만들고 새로운 기능을 추가해서 오버라이딩할 수 있겠지만, 상속을 사용할 수 없는 제한적인 상황이 있을 수도 있고, 상속으로 인해 단일 책임 원칙에 어긋나게 되는 경우도 있을 것이다.  
(std::vector를 상속받을 경우 vector에는 버추얼 소멸자가 없기때문에 문제가 있을 수 있다.)
